Tipos Genéricos en Java

1. ¿Qué son los Tipos Genéricos?

Los tipos genéricos permiten escribir clases, interfaces y métodos que pueden trabajar con diferentes tipos de datos mientras mantienen la seguridad de tipos en tiempo de compilación.

Ventajas principales:
  • Seguridad de tipos en tiempo de compilación
  • Eliminación de casteos explícitos
  • Permiten implementar algoritmos genéricos
  • Reutilización de código
  • Mayor robustez en el código

2. Sintaxis Básica

Declaración de una clase genérica:

public class Contenedor<T> {
    private T contenido;
    
    public void setContenido(T contenido) {
        this.contenido = contenido;
    }
    
    public T getContenido() {
        return contenido;
    }
}

Convenciones de nomenclatura comunes:

  • T - Tipo general (Type)
  • E - Elemento (Element)
  • K - Clave (Key)
  • V - Valor (Value)
  • N - Número (Number)

3. Uso de Tipos Genéricos

Ejemplo básico:

// Crear instancia con tipo específico
Contenedor<String> contenedorString = new Contenedor<>();
contenedorString.setContenido("Hola Mundo");
String contenido = contenedorString.getContenido(); // No necesita casting

// Otro tipo
Contenedor<Integer> contenedorInt = new Contenedor<>();
contenedorInt.setContenido(42);
int numero = contenedorInt.getContenido();

Múltiples parámetros de tipo:

public class Par<K, V> {
    private K clave;
    private V valor;
    
    public Par(K clave, V valor) {
        this.clave = clave;
        this.valor = valor;
    }
    
    public K getClave() { return clave; }
    public V getValor() { return valor; }
}

4. Restricciones de Tipos (Bounds)

Upper Bounded Wildcards:

// Acepta Number y sus subclases
public void procesarNumeros(List<? extends Number> lista) {
    for (Number n : lista) {
        System.out.println(n.doubleValue());
    }
}

Lower Bounded Wildcards:

// Acepta Integer y sus superclases
public void agregarEnteros(List<? super Integer> lista) {
    lista.add(42);
}

Restricción de tipo:

// T debe ser subclase de Number
public class Calculadora<T extends Number> {
    public double sumar(T a, T b) {
        return a.doubleValue() + b.doubleValue();
    }
}

5. Métodos Genéricos

Método genérico simple:

public static <T> void imprimirArray(T[] array) {
    for (T elemento : array) {
        System.out.println(elemento);
    }
}

Método con múltiples tipos:

public static <T, U> Par<T, U> crearPar(T primero, U segundo) {
    return new Par<>(primero, segundo);
}

Método con restricción de tipo:

public static <T extends Comparable<T>> T encontrarMaximo(T[] array) {
    if (array == null || array.length == 0) return null;
    
    T max = array[0];
    for (int i = 1; i < array.length; i++) {
        if (array[i].compareTo(max) > 0) {
            max = array[i];
        }
    }
    return max;
}

6. Wildcards

Wildcard sin restricción:

public void procesarLista(List<?> lista) {
    for (Object elemento : lista) {
        System.out.println(elemento);
    }
}

Wildcard con límite superior:

public double sumarLista(List<? extends Number> lista) {
    double suma = 0.0;
    for (Number numero : lista) {
        suma += numero.doubleValue();
    }
    return suma;
}

Wildcard con límite inferior:

public void llenarLista(List<? super Integer> lista) {
    for (int i = 1; i <= 5; i++) {
        lista.add(i);
    }
}

7. Ejemplos Prácticos

Ejemplo 1: Stack Genérico

public class Stack<T> {
    private ArrayList<T> elementos = new ArrayList<>();
    
    public void push(T elemento) {
        elementos.add(elemento);
    }
    
    public T pop() {
        if (elementos.isEmpty()) {
            throw new EmptyStackException();
        }
        return elementos.remove(elementos.size() - 1);
    }
    
    public boolean isEmpty() {
        return elementos.isEmpty();
    }
}

Ejemplo 2: Par Ordenado

public class ParOrdenado<T extends Comparable<T>> {
    private T primero;
    private T segundo;
    
    public ParOrdenado(T primero, T segundo) {
        if (primero.compareTo(segundo) <= 0) {
            this.primero = primero;
            this.segundo = segundo;
        } else {
            this.primero = segundo;
            this.segundo = primero;
        }
    }
    
    public T getPrimero() { return primero; }
    public T getSegundo() { return segundo; }
}

8. Mejores Prácticas

  • Usar nombres descriptivos para los parámetros de tipo
  • Limitar el uso de wildcards a APIs públicas
  • Preferir tipos específicos sobre wildcards cuando sea posible
  • Documentar el propósito de cada parámetro de tipo
  • Considerar las restricciones de tipo para garantizar operaciones válidas
Consideraciones importantes:
  • Los tipos genéricos se borran en tiempo de ejecución (type erasure)
  • No se pueden crear arrays de tipos genéricos
  • No se pueden usar tipos primitivos como argumentos de tipo (usar wrappers)
  • No se pueden crear instancias de tipos genéricos
