Clases Anónimas en Java

1. ¿Qué son las Clases Anónimas?

Las clases anónimas son clases sin nombre que se declaran e instancian al mismo tiempo. Son una forma de crear una subclase o implementar una interfaz de manera inmediata sin necesidad de crear una clase separada.

Características principales:
  • No tienen nombre
  • Se declaran y crean en una sola expresión
  • Son clases internas
  • Solo pueden implementar una interfaz o extender una clase
  • Tienen acceso a las variables finales o efectivamente finales del contexto que las rodea

2. Sintaxis Básica

Implementando una interfaz:

Interface miInterfaz = new Interface() { @Override public void miMetodo() { // Implementación } };

Extendiendo una clase:

MiClase objeto = new MiClase() { @Override public void miMetodo() { // Nueva implementación } };

3. Ejemplos Prácticos

Ejemplo 1: Runnable

Runnable runnable = new Runnable() { @Override public void run() { System.out.println("Ejecutando en un hilo"); } }; // Usar el runnable new Thread(runnable).start();

Ejemplo 2: ActionListener

JButton boton = new JButton("Clic"); boton.addActionListener(new ActionListener() { @Override public void actionPerformed(ActionEvent e) { System.out.println("Botón presionado"); } });

Ejemplo 3: Comparator

List lista = new ArrayList<>(); Collections.sort(lista, new Comparator() { @Override public int compare(String s1, String s2) { return s1.length() - s2.length(); } });

4. Casos de Uso Comunes

Listeners de Eventos

// Ejemplo con MouseListener component.addMouseListener(new MouseAdapter() { @Override public void mouseClicked(MouseEvent e) { System.out.println("Clic en: " + e.getX() + "," + e.getY()); } @Override public void mouseEntered(MouseEvent e) { System.out.println("Mouse entró"); } });

Thread personalizado

Thread thread = new Thread() { @Override public void run() { for(int i = 0; i < 5; i++) { System.out.println("Contador: " + i); try { Thread.sleep(1000); } catch(InterruptedException e) { e.printStackTrace(); } } } }; thread.start();

Callbacks personalizados

interface Callback { void onComplete(String result); } public void procesarAsincrono(final Callback callback) { new Thread(new Runnable() { @Override public void run() { // Proceso largo String resultado = "Proceso completado"; callback.onComplete(resultado); } }).start(); }

5. Acceso a Variables Externas

Las clases anónimas pueden acceder a:
  • Variables finales del método que las contiene
  • Variables efectivamente finales (no modificadas después de su inicialización)
  • Variables de instancia y métodos de la clase que las contiene
public class EjemploAcceso { private int variableInstancia = 1; public void metodo() { final int variableFinal = 2; int variableEfectivamenteFinal = 3; Runnable r = new Runnable() { @Override public void run() { System.out.println(variableInstancia); // OK System.out.println(variableFinal); // OK System.out.println(variableEfectivamenteFinal); // OK } }; } }

6. Ejemplos Avanzados

Ejemplo 1: Factory Method con clase anónima

public interface Producto { void usar(); } public class Factory { public Producto crearProducto(final String tipo) { return new Producto() { private final String tipoProducto = tipo; @Override public void usar() { System.out.println("Usando producto: " + tipoProducto); } }; } }

Ejemplo 2: Estado interno en clase anónima

public class Contador { public Runnable crearContador() { return new Runnable() { private int cuenta = 0; @Override public void run() { System.out.println("Cuenta: " + (++cuenta)); } }; } }

7. Comparación con Lambdas

Clases Anónimas vs Lambdas:
  • Las clases anónimas pueden tener estado (variables de instancia)
  • Pueden implementar múltiples métodos
  • Pueden tener constructores
  • Son más verbosas que las lambdas
  • Pueden extender clases (las lambdas no)
// Clase anónima Runnable r1 = new Runnable() { @Override public void run() { System.out.println("Clase anónima"); } }; // Lambda equivalente Runnable r2 = () -> System.out.println("Lambda");

8. Mejores Prácticas

  • Usar lambdas en lugar de clases anónimas para interfaces funcionales
  • Mantener las clases anónimas pequeñas y enfocadas
  • Considerar crear una clase nombrada si la lógica es compleja
  • Documentar el propósito de la clase anónima
  • Evitar anidar múltiples clases anónimas
Cuándo usar clases anónimas:
  • Cuando necesitas mantener estado
  • Cuando necesitas implementar múltiples métodos
  • Cuando necesitas extender una clase
  • En situaciones donde la legibilidad es más importante que la concisión

9. Consideraciones de Rendimiento

  • Cada clase anónima crea un nuevo archivo .class
  • Mayor consumo de memoria que las lambdas
  • Tiempo de carga de clase adicional
  • Pueden impactar en el tiempo de inicio de la aplicación
